# 2026-04-14 — Update Sử dụng MLX và model Qwen3.5-4B

**Version:** 1.1.1  
**Type:** Infrastructure  
**Scope:** Infrastructure | Docs

---

## Summary
Migrated the inference engine from a Dockerized `llama-cpp` deployment to a native MLX deployment running on the Apple Silicon macOS Host. This shift allows the platform to fully utilize Metal GPU acceleration for the `Qwen3.5-4B` model, vastly improving inference speed that was previously bottlenecked by the Docker VM.

---

## Added
* Added `scripts/start_mlx.sh` to initialize a Python virtual environment, install `mlx-lm`, and start the MLX inference server natively on the host binding to `0.0.0.0:9999`.

## Changed
* Updated `docker-compose.yml` to point backend `VLLM_BASE_URL` to `http://host.docker.internal:9999/v1` instead of the local Docker network `llama-cpp` service.
* Updated `docs/architecture.md` diagram and service inventory to reflect the new MLX-on-host topography.

## Fixed
* Bypassed the limitation where macOS Docker Desktop cannot pass the GPU to Linux containers.

## Removed
* Removed `llama-cpp` service from `docker-compose.yml`.
* Removed `model-downloader` service from `docker-compose.yml`.
